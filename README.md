# Практическая работа № 2.1

**Исследование возможностей чат-ботов с искусственным интеллектом для решения задач добывающей промышленности (на примере сравнительного анализа кейсов)**

**Автор:** Булгаков А.А.  
**Группа:** БНГА-о24  
**ЮРГПУ (НПИ)** 

## Цель работы
Провести сравнительный анализ функциональности зарубежного и отечественного чат-бота с искусственным интеллектом. Смоделировать сценарии использования для горнодобывающего или нефтегазового сектора и оценить эффективность работы каждого из ботов. Результаты опубликовать в открытом репозитории GitHub.

## Выбранные чат-боты
* **Зарубежный:** Gemini (Google)
* **Отечественный:** YandexGPT (Алиса)

## Выбранные кейсы
1. Диагностика бурового насоса НБ-125 (падение давления масла, перегрев корпуса, повышенная вибрация).
2. Анализ требований безопасности при эксплуатации буровых установок (действия при ГНВП и подбор СИЗ).

## Ход работы (по методичке)
1. Выбраны представители зарубежных (Gemini) и отечественных (YandexGPT) LLM-чат-ботов.
2. Выявлены ключевые проблемные области: техническая диагностика оборудования и производственная безопасность в бурении.
3. Разработаны одинаковые запросы для решения задач в рамках кейсов.
4. Проведено тестирование, зафиксированы результаты работы обоих ботов (в папке `results/`).
5. Подготовлен отчёт с анализом и выводами.

## Методические указания (что выполнено)
* Использованы два сервиса: зарубежный и отечественный чат-бот.
* Кейсы выбраны из профильных направлений (ремонт оборудования и промышленная безопасность).
* Запросы отправлены в новых чатах без истории.
* Результаты сохранены в виде скриншотов.

## Структура репозитория
* `cases/` — описание кейсов и запросов
* `prompts/` — тексты запросов
* `results/` — ответы ботов (скриншоты)
* `analysis/` — сравнительный анализ и выводы
* `README.md` — этот документ

## Как посмотреть
Все диалоги и результаты — в папке `results/`:
* Файлы с "Gemini" — ответы зарубежного бота.
* Файлы с "Алиса" — ответы отечественного бота.

Сравнительный анализ и выводы: `analysis/comparative_analysis.md`
